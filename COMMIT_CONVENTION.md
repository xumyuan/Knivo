# Commit Convention

格式：

```
<type>: <简短描述>

<可选的详细说明>
```

## Type 类型

| 类型       | 用途                 |
| ---------- | -------------------- |
| `init`     | 项目初始化           |
| `feat`     | 新功能               |
| `fix`      | 修复 bug             |
| `refactor` | 重构（不改变功能）   |
| `docs`     | 文档变更             |
| `style`    | 格式调整（不影响逻辑）|
| `test`     | 测试相关             |
| `chore`    | 构建、依赖等杂项     |

## 规则

- 描述使用中文或英文均可，保持简洁
- 首行不超过 72 个字符
- 详细说明可用 `-` 列出要点
